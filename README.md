# CodeStyle for Base C++ homework
Ниже приведен codestyle в соответсвии которого необходимо писать домашнее задание.
1. В коде нельзя использовать символ табуляции, необходимо использовать только пробелы.

2. Оператор открывающаяся фигурная скобочка `{` должна быть на той же строке оператора цикла, или функции. Также между оператором фигурная скобочка и оператором должен быть пробел
```
Например
int main() {
    int a = 0;
    if ( a == 0 ) { // важно после if и между { пробел
    }
}
```

3. Начало инструкции относительного блока должна начинаться с четырёх пробелов.
```
if ( a == 0 ) {
  a = 10; // относительно оператора if и инструкцией a = 10 четыре пробела
}
```

4. Между операторами должен быть всегда пробел
```
int a = c + b + d;

...
if ( a == 10 ) { // как видно между оператором == стоит пробел с обоих сторон
  ....
}
```

5. Новая инструкция всегда должна начинаться с новой строки
```
if ( a == 10 ) ++a; // это неправильно

if ( a == 10 ) {
  ++a;
}
```

6. Название переменных должно быть в стиле camelCase(case_style) с маленькой буквы
```
int myVariable = 100;

int countTestCode = 10;
```

7. Не использовать `using namespace std`
8. Константы должны быть в верхнем регистре в `SNAKE_CASE` стиле
9. Ветка `else` и ` else if ` должна начинаться на той же строчке где и закрывающая скобка `}`
10. Для именования используйте полные названиия, а не сокращения

```с++
int sourceColumn{ 0 }; // слова без сокращений -- правильно
std::string tcpSocket; // "tcp" является стандартным сокращением -- правильно
```

11. Переменные члены класса именуются с суффиксом _. Это позволяет отличать их от локальных переменных.

12. Глобальные переменных именуются с суффиксом _g. Это позволяет отличать их от локальных переменных и от переменных членов класса.

13. Статические переменные именуются с суффиксом _s. Это позволяет отличать их от локальных переменных, глобальных переменных и от переменных членов класса ( статические переменные внутри функций именуются как обычные локальные переменные ).

14. Типы именуются в CamelCase стиле и начинаются с большой буквы. Под типами подразумеваются: classes, structures, enums, unions, template parameters, typedefs.
